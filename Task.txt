Оценка практической работы:
Веб сервер стартует и отвечает на запросы из браузера +

На странице регистрации пользователь вводит логин, пароль и email и сервер запоминает  на время своей работы (без сохранения в базу). +

При совпадении логина с уже существующим, пользователь получает сообщение об ошибке +

Зарегестрированный  пользователь может залогиниться по своему логину и паролю. +

При ошибке в пароле пользователь получает сообщение об ощибке. +

После логина, при запросе любых страниц сервер должен узнавать пользователя. +

Логин и email текущего пользователя можно посмотреть на странице профиля. +

При  запросе профиля не авторизованным пользователем – он должен быть перенаправлен на страницу авторизации. +

При заходе второго пользователя (с другого браузера) он  должен проходить регистрацию и авторизацию никак не влияя на первого. - ? (как помешать-то можно?)

Пользователь должен иметь возможность выйти из приложения. -

Сервер должен получать порт для старта из параметров запуска. -

Сервер должен содержать сервлет для администрирования. Со страницы администрирования должна быть возможность остановить сервер и посмотреть количество зарегестрированных и залогиненных пользователей. -
